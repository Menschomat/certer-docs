The provider provides three core resources to manage your Certer state.

---

## 1. `certer_team`

Manages a tenant team.

```hcl
resource "certer_team" "devops" {
  name        = "devops-team"
  description = "Infrastructure core services"
}
```

### Argument Reference
*   `name` (String, Required): Unique identifier name of the team.
*   `description` (String, Optional): Explanation of team purpose.

### Attribute Reference
*   `id` (String): The UUIDv7 generated by the server.

---

## 2. `certer_certificate`

Defines domain certificates that Certer should fetch and keep renewed.

```hcl
resource "certer_certificate" "wildcard" {
  primary = "example.com"
  team_id = certer_team.devops.id
  sans    = [
    "*.example.com",
    "www.example.com"
  ]
}
```

### Argument Reference
*   `primary` (String, Required): The main certificate domain name. (Triggers replacement if changed).
*   `team_id` (String, Required): The owning team UUID.
*   `sans` (List of Strings, Optional): Subject Alternative Names.

### Attribute Reference
*   `id` (String): Unique certificate UUIDv7.

---

## 3. `certer_api_key`

Creates client and admin API keys.

```hcl
resource "certer_api_key" "fetch_token" {
  description          = "client-fetcher"
  allowed_certificates = [certer_certificate.wildcard.id]
  allowed_teams        = [certer_team.devops.id]
  admin                = false
}

# Plaintext token output
output "client_token" {
  value     = certer_api_key.fetch_token.cleartext_token
  sensitive = true
}
```

### Argument Reference
*   `description` (String, Optional): User-friendly description of token purpose.
*   `admin` (Boolean, Required): If `true`, grants administrative control. If `false`, grants access only to fetch PEM files.
*   `allowed_teams` (List of Strings, Optional): Limit API key access to target teams.
*   `allowed_certificates` (List of Strings, Optional): Limit token access to target certificates.

### Attribute Reference
*   `id` (String): Key UUIDv7.
*   `cleartext_token` (String, Sensitive): The cleartext token. **Only readable in Terraform state on creation**.

---

## API Key Scoping Matrix

Use the `admin` flag combined with scoping arrays to secure credentials:

| Key Type | `admin` | `allowed_teams` | Scope / Capabilities |
|---|---|---|---|
| **Root Admin** | `true` | `[]` *(Empty)* | Absolute CRUD controls across all teams and configurations. |
| **Scoped Admin** | `true` | `["team_uuid"]` | Configure certificates and keys ONLY inside the allowed teams. |
| **Fetch Client** | `false` | `["team_uuid"]` | Only allowed to fetch raw certificate assets. No configuration changes. |
